平常除了看診，我也寫文章、錄 Podcast，最近還打算寫書。但打字一直是我的瓶頸——腦袋轉得比手指快太多，常常一個想法還沒打完，下一個就溜走了。

所以我很早就開始找語音輸入工具。

## 試過的方案，都差一點

最早用的是 Typeless，體驗確實不錯。後來因為隱私疑慮換成 Wisprflow，但安裝時碰到一些讓人不舒服的狀況，而且它在 Android 上會常駐背景，總有種「手機被監聽」的不安感。

開源方案也看了不少，都不太滿意——要麼只有 Mac 版，要麼用 Python 打包，桌面上資源佔用很大。

身為一個也喜歡寫程式的人，我決定：自己做一個。

## 從沒寫過 App，但我知道我要什麼

先說我的背景。我熟悉 Web 開發——Python、JavaScript（React）、簡單的 SQL，平常也用 Git 和 GitHub。但我從來沒寫過 Android App，也從來沒做過 Windows 桌面應用。

以前要從頭開發網站、Android 版、Windows 版再加上測試，大概要好幾個月。

但這次我用了 [Claude Code](https://claude.ai/claude-code)。

Claude Code 的能力超乎我的想像。我雖然不熟 Android 和桌面開發，但我清楚自己要什麼、知道該往哪個方向走，Claude Code 負責幫我執行。從發想到第一個可用版本，只花了幾個小時。之後持續邊測邊改，加入語音編輯、翻譯、語調切換（隨和 vs. 正式）、App 間無縫切換等功能。經過好幾天的日常使用和反覆測試，覺得真的實用了，就決定開源。

## 技術選擇：為什麼是 Tauri 和 Groq

桌面版我選了 [Tauri](https://tauri.app/)（基於 Rust），而不是常見的 Electron 或 Python 方案。原因很直接：安裝檔極小、資源佔用極低，而且原生支援 Windows、Mac、Linux。我自己主要在 Windows 上用，Mac 版請朋友測試過（但還沒成功……），Linux 版坦白說也還沒充分實測——歡迎 Linux 用戶回報！

Android 版設計成「輸入法」形式：需要時一鍵切過來，用完一鍵切回去，不會常駐背景。中英混打也沒問題，App 會自動整理成完整句子。

語音辨識的部分，我本來想用本地端的 Whisper 模型，但 whisper-rs 在 Windows 上有相容性問題。後來找到 [Groq](https://groq.com/) 提供的免費 API，速度快到幾乎即時，背後的語音模型和 LLM 都是開源的。我認為語音辨識後的語意修正不需要超大模型，關鍵是速度——讓「想到什麼就說什麼」的流暢感不被打斷。

## 用語音寫程式的迴圈

這整篇文章，就是用 VoxPen 口述完成的。

說一個有趣的事：現在我用 Claude Code 寫程式的時候，甚至可以直接用說的，連打字都不用。一個精神科醫師，靠語音寫出了語音輸入工具，然後再用這個工具語音寫程式。

聽起來沒什麼了不起的，但這就是 2026 年的開發體驗。

## 隱私優先，開源透明

VoxPen 採 BYOK（Bring Your Own Key）架構——你的語音資料直接送到你自己的 API，不經過我的伺服器。免費版有每日使用次數上限，帶入自己的 API Key 則無限制。

程式碼已經開源放在 [GitHub](https://github.com/soanseng/voxpen-desktop) 上。

如果你也受夠了打字的慢，或是你是開發者想一起貢獻，歡迎[安裝試用](/zh-tw/download)。
